1. Choose a different name for the application that is not confusingly similar to "1Key"
2. Replace the application icon with one that is not confusingly similar to the 1Key icon
3. Update visible identifiers (app title, About screen, package name where reasonable) to reflect the new name
4. Retain the GPL-3.0 license file and the original copyright attribution as required by the GPL

This is the standard pattern used by Mozilla (Firefox -> Iceweasel), Chromium-based forks, and other GPL projects with a distinct brand identity.

## Why this exists

The GPL guarantees that anyone can run, study, modify, and redistribute the code. It does not guarantee the right to confuse users about which "1Key" they are downloading. Section 7(e) of the GPL-3.0 explicitly permits this kind of trademark-related restriction as a non-conflicting additional term.

The combined effect of GPL-3.0 plus this notice is that:

- A fork that fixes bugs and adds features under a different name is legally permitted but is not endorsed by the maintainer; users installing such a fork do so without the maintainer's recommendation
- A fork that ships a closed-source "1Key Pro" with added telemetry is blocked by the GPL
- A fork that ships an open-source "1Key Pro" trading on the original brand is blocked by this notice
